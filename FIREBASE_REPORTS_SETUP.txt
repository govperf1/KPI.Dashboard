GRC REPORT UPLOAD — ONE-TIME FIREBASE SETUP
============================================

The platform now uploads PDF reports directly from the Super Admin account to Firebase Storage,
and stores report metadata in the Firestore collection: grc_reports.

IMPORTANT
---------
1) Cloud Storage for Firebase must be enabled for this project.
2) The project must use the Blaze plan for Cloud Storage access.
3) Merge the following rules into your existing rules. Do NOT delete your other project rules.

FIRESTORE RULE TO ADD
---------------------
Inside: service cloud.firestore { match /databases/{database}/documents { ... } }

match /grc_reports/{reportId} {
  allow read: if request.auth != null;
  allow create, update, delete: if request.auth != null
    && get(/databases/$(database)/documents/users/$(request.auth.token.email)).data.approved == true
    && get(/databases/$(database)/documents/users/$(request.auth.token.email)).data.role == 'super_admin';
}

STORAGE RULE TO ADD
-------------------
Inside: service firebase.storage { match /b/{bucket}/o { ... } }

match /grc_reports/{allPaths=**} {
  allow read: if request.auth != null;
  allow create, update: if request.auth != null
    && firestore.get(/databases/(default)/documents/users/$(request.auth.token.email)).data.approved == true
    && firestore.get(/databases/(default)/documents/users/$(request.auth.token.email)).data.role == 'super_admin'
    && request.resource.contentType == 'application/pdf'
    && request.resource.size < 100 * 1024 * 1024;
  allow delete: if request.auth != null
    && firestore.get(/databases/(default)/documents/users/$(request.auth.token.email)).data.approved == true
    && firestore.get(/databases/(default)/documents/users/$(request.auth.token.email)).data.role == 'super_admin';
}

After publishing the rules, sign in as Super Admin, open GRC > Reports, and use “Upload Report”.
